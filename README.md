# WeChat Auto Workflow

面向微信公众号内容生产的 AI 工作流模块集合。它不绑定某一个 AI 客户端或运行平台，可以被支持读取模块说明、调用脚本和执行工具链的 AI 助手、自动化代理或本地流程复用。

项目覆盖从选题写作、AI 写作风险审核、配图调研，到微信 HTML 排版预览的内容生产链路。默认强调人工可控、来源可追溯和复制可用：交付本地预览页或可复制 HTML，由使用者在微信后台完成后续发布操作。

## 能解决什么

- 策划、写作或改写微信公众号文章。
- 生成微信友好的 `article#wechatArticle` HTML 正文。
- 生成本地预览页，方便复制到微信后台。
- 审核文章的 AI 写作风险和低信息密度问题。
- 为正文和封面调研可追溯图片候选。

## 模块结构

```text
skills/
  wechat-article-pipeline/      # 完整流水线入口
  wechat-article-writer/        # 公众号写作与改写
  wechat-article-formatter/     # HTML 排版、主题化和预览
  article-image-research/       # 配图调研、授权和风险初筛
  ai-writing-risk-review/       # AI 写作风险评估
dist/                           # 本地预览和示例产物，默认不作为源码发布
```

目录名 `skills/` 和文件名 `SKILL.md` 是为了兼容常见 AI 助手的能力模块格式；项目本身不依赖 Codex，也不要求只能在 Codex 中运行。其他代理或工作流系统可以把这些 Markdown 文件当作流程规范，把 `scripts/` 中的 Python 脚本当作可执行工具。

## 推荐入口

完整文章生产流程使用：

```text
wechat-article-pipeline
```

它会按顺序编排：

```text
主题/资料
  -> 写作策略与大纲
  -> HTML 主稿
  -> AI 写作风险审核
  -> 配图调研
  -> 样式审核与预览
  -> 本地预览或可复制 HTML
```

如果只需要某一环节，可以单独使用对应模块：

- `wechat-article-writer`：只处理写作、改写、标题、摘要和结构。
- `wechat-article-formatter`：只处理 HTML 正文、主题样式和本地预览。
- `article-image-research`：只处理配图搜索、授权归因和风险初筛。
- `ai-writing-risk-review`：只评估 AI 写作、人机混写或模板化风险。

## 安全边界

- 项目当前只负责内容生成、排版预览和复制准备。
- 本地图片会在预览和复制场景中尽量内联为 Base64 Data URL，降低复制到微信后台时的失效风险。
- 未核验事实、图片版权、肖像权或敏感内容需要在使用前人工复核。
- 生成结果应先在本地预览页检查，再复制到微信后台。

## 对外定位

这个项目适合被描述为：

> 一套平台无关的微信公众号 AI 内容工作流模块，从写作、审核、配图到排版预览，帮助创作者把“生成文章”变成可复查、可预览、可复制的生产流程。
