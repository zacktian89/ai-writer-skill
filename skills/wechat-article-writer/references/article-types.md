# 文章类型

选择最接近的一类，再按任务微调。

## 1. 热点评论 / 时评

目标：

- 靠判断获得打开
- 靠立场、信息和解释获得转发

结构：

1. 先给判断。
2. 说明多数人漏看了什么。
3. 给 2 到 4 条证据或推理。
4. 回应一个明显反对意见。
5. 落到更大的影响或下一步观察点。

注意：

- 不要用前三分之一复述新闻。
- 价值在角度和判断，不在复述。
- 语气要有分寸。事实没核验时，不要把推测写成定论。

## 2. 干货教程 / 方法文

目标：

- 靠明确收益获得打开
- 靠步骤清楚获得读完
- 靠可复用获得收藏

结构：

1. 用读者的话说出问题。
2. 预告方法或框架。
3. 按顺序解释步骤。
4. 加一个例子、误区或边界条件。
5. 结尾给下一步行动或检查表。

注意：

- 小标题尽量像读者会问的问题。
- 读者二次阅读时，应该能直接按步骤操作。
- 不要把概念解释写成资料堆叠。

## 3. 行业分析 / 趋势解读

目标：

- 靠相关性获得打开
- 靠解释力获得读完
- 靠判断框架获得转发

结构：

1. 发生了什么变化。
2. 为什么会发生。
3. 影响谁。
4. 接下来可能怎样演化。
5. 读者应该看什么、做什么或避开什么。

注意：

- 每个事实都要连接到真实影响。
- 不要把文章写成一堆互不相连的信息。
- 重要判断要放在解释前面，减少绕路。

## 4. 故事 / 情感 / 人物

目标：

- 靠张力和共情获得打开
- 靠场景推进获得读完
- 靠情绪完成获得分享

结构：

1. 从场景、选择、对话或冲突开始。
2. 背景只在需要时补充。
3. 每个部分推动关系、命运或认知变化。
4. 意义提炼靠后出现。
5. 结尾停在一个画面、选择或醒悟上。

注意：

- 不要在读者感受到情绪前先解释情绪。
- 少写抽象感受，多写可观察细节。
- 不要在结尾硬加道理。

## 5. 清单 / 盘点 / 合集

目标：

- 靠范围和筛选价值获得打开
- 靠便利和可复用获得收藏

结构：

1. 说明筛选标准。
2. 按清晰顺序呈现条目。
3. 每个条目给一个有用判断。
4. 必要时比较取舍。
5. 按读者类型给推荐。

注意：

- 没有筛选逻辑的清单像搬运。
- 排名、分组、排除标准本身就是价值。
- 条目之间的写法要统一，但不要机械重复。

## 6. 通知 / 政策 / 服务信息

目标：

- 靠个人相关性获得打开
- 靠清晰行动获得读完

结构：

1. 谁受影响。
2. 发生了什么变化。
3. 什么时候生效。
4. 读者需要做什么。
5. 去哪里查看官方详情。

注意：

- 先翻译成读者行动，再附原始通知。
- 不要把原公告语言直接放在文章开头。
- 重要时间、对象、条件必须核验。
