# 微信公众号写作

## 用途

用这个技能写作或改写微信公众号文章。目标是让文章更容易被打开、读完、保存、转发，同时保持准确、克制、有中文语感。

这个技能只处理写作质量和文章策略。

- 需要系统性查找、筛选和归因配图时，使用 `article-image-research`。
- 需要生成微信可复制 HTML 或预览时，使用 `wechat-article-formatter`。

## 核心规则

> **5 条永远遵守的规则：**
> 1. **先确认再写**——大纲和标题必须经用户确认后才起草正文。
> 2. **先约束再写**——字数预算、各节任务、禁写内容必须在起草前确定。
> 3. **标题兑现**——正文必须兑现标题承诺，不透支账号信任。
> 4. **信息推进**——每段必须推进新信息，删除纯承接段。
> 5. **事实标注**——未核验的时效/政策/数据内容必须标注 `[待核验]`。

## 硬性门控

违反任何一条必须修正后再交付：

1. 起草正文前，必须将选题承诺、结构大纲和标题备选反馈给用户并获得确认。**未确认不得起草。**
2. 成稿字数必须在写前设定的目标篇幅 **±15%** 以内。超出时必须先裁剪，不得直接交付。
3. 每个正文小节必须推进至少一项：事实、判断、场景、方法或情绪。**不推进任何新信息的小节必须删除或合并。**
4. 标题承诺必须能在正文中找到对应兑现段落。**无法兑现时必须修改标题。**
5. 涉及时效事实、政策、数据、医疗/法律/金融建议时，未核验的内容必须标注 `[待核验]`。**不得将推测写成定论。**

## 执行清单

开始写作或改写后用这张清单跟踪进度：

- [ ] 明确文章目标、目标读者、账号背景和交付形态。
- [ ] 判断文章类型，按需读取 `references/article-types.md`。
- [ ] 写文章承诺、确定主声口、立场边界和精简约束。详见 `references/writing-workflow.md`。
- [ ] 设计标题（至少 3 个备选）、摘要、开头、正文结构和结尾动作。
- [ ] **[门控]** 将大纲框架和标题备选反馈给用户，获得确认后再继续。
- [ ] 写作或改写正文，按写前约束控制篇幅和信息密度。
- [ ] 用 `references/review-checklist.md` 做最终检查，确认符合硬性门控。

## 写作优先级

当规则发生冲突时，按以下顺序取舍：

1. 准确和可信优先于传播技巧。
2. 清楚立场优先于完整铺陈。
3. 具体的人、事、场景优先于抽象概念。
4. 自然声口优先于整齐结构。
5. 有记忆点的一处细节优先于平铺直叙的全面覆盖。

不要为了满足所有检查项，把文章写成没有脾气、没有现场、没有取舍的说明文。

## 工作流程

详细步骤和模板见 `references/writing-workflow.md`。高层流程：

1. **定位**：判断文章目标、类型（见 `references/article-types.md`）、目标读者和主声口。
2. **承诺与立场**：写文章承诺，有观点的文章先写立场卡（判断、理由、边界）。
3. **约束**：按 `references/length-guidelines.md` 设定字数预算，确定各节任务、保留内容和禁写内容。
4. **框架**：设计标题（至少 3 个，参考 `references/headline-patterns.md`）、摘要、开头、正文结构和结尾。
5. **[门控] 确认**：将承诺、大纲、标题反馈给用户，获得确认。
6. **起草**：按约束写正文。只插入能支撑具体段落的图片。基于网页资料时按 `references/image-research.md` 记录图片候选；需要系统性找图时调用 `article-image-research`。
7. **检查**：用 `references/review-checklist.md` 做最终检查，确认符合硬性门控。

## 默认策略

- 默认先写清楚"读者读完会多知道什么"，再决定标题和结构。
- 默认保留用户给出的事实、观点、口吻限制和专有名词；改写时只动表达、节奏和结构。
- 默认不编造数据、案例、出处、人物经历或政策细节；缺资料时标注 `[待核验]`。
- 默认标题给至少 3 个选项，正文只采用一个主标题策略。
- 默认写现代简体中文。除非用户要求翻译或保留原文，不要改变语种。
- 默认采用紧凑表达：先写结论，再给理由；能一句说清的，不扩成一段。
- 默认把资料来源、图片候选、核验备注放在交付说明，不写成正文固定章节。
- 默认只消费已确认的图片候选；需要主动找图时交给 `article-image-research`。
- 默认不生成微信 HTML；需要排版时交给 `wechat-article-formatter`。

## 中文写作基线

- 一段只讲一个中心意思。中心句尽量靠前。
- 一个句子尽量只推进一层意思。遇到多重修饰、多个因果或多个转折，拆句。
- 多用具体名词、动作、场景、结果，少用抽象概念堆叠。
- 先说判断结论或核心信息，再解释理由。读者需要先知道作者站在哪里。
- 每 300 到 500 个汉字给一次信息回报：结论、案例、数字、反差、可执行步骤或关键提醒。
- 删掉不能回答"读者读完多知道了什么"的句子，尤其是重复铺垫、同义改写、泛泛转折和结尾复述。
- 小标题要能单独被扫读。只看小标题，也应能理解文章主线。
- 结尾不能只复述正文，要给读者一个保存、转发、讨论或行动的理由。

## 常见误用

- 不要把标题写得比正文更夸张；标题承诺必须能在正文里兑现。
- 不要为"去 AI 腔"刻意加入错别字、口水话或无意义的个人感受。
- 不要用整齐的小标题和万能转折代替真实判断。
- 不要把所有参考文件一次性读完；只按任务需要读取相关文件。
- 不要在没有核验的情况下写确定性时效判断、政策判断、医疗法律金融建议或具名争议结论。
