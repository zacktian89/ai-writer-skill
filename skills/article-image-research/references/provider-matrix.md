# 图片来源矩阵

## 选择顺序

1. 用户素材和本地素材：最适合品牌、活动、内部案例、真实场景。
2. 原文或引用资料图片：最适合翻译、综述、技术讲解、报告解读。
3. 官方来源：最适合产品、政策、机构、数据、版本和公开事件。
4. Wikimedia Commons / Openverse：最适合开放授权、百科类、历史、地点、人物、机构、公共资料。
5. Pexels / Unsplash：最适合场景图、氛围图、抽象主题图、封面候选。
6. Brave / SerpApi / Google Images：最适合发现特定对象或线索，但必须二次核验授权和来源。
7. AI 生成图片：最适合概念化封面、抽象场景、非事实示意图；优先级低于可核验的搜索结果。

## Openverse

- 适合：开放授权图片、公共领域图片、需要归因的正式文章。
- 优点：返回作者、license、license_url、provider、落地页、attribution 等字段。
- 风险：Openverse 聚合第三方元数据，授权准确性仍需在落地页核验。
- 使用：优先筛选 `cc0`、`pdm`、`by`，慎用限制较多或需要相同方式共享的授权。

## Wikimedia Commons

- 适合：人物、地点、历史、机构、公共事件、文物、地图、公开资料。
- 优点：文件页通常包含作者、授权、来源、使用历史和结构化数据。
- 风险：授权类型复杂，部分图片有肖像权、商标、建筑物权利或国家差异。
- 使用：保存文件页 URL，不只保存 `upload.wikimedia.org` 图片直链。

## Unsplash

- 适合：高质量摄影、开头图、封面候选、抽象商业主题、场景氛围。
- 优点：图片质量高，可按关键词、方向、颜色、分页筛选。
- 风险：API 使用要求归因和下载事件追踪；照片不等于事实证据。
- 使用：保留摄影师、Unsplash 落地页和 API 返回的归因信息。

## Pexels

- 适合：高质量摄影、场景图、封面候选、视频素材线索。
- 优点：可搜索照片和视频，通常不强制归因。
- 风险：仍需注意人物肖像、商标、建筑、敏感场景和二次分发限制。
- 使用：保留摄影师、Pexels 落地页和图片 ID。

## Brave / SerpApi / Google Images

- 适合：查找特定事件现场、产品界面、新闻图片、官方页面线索。
- 优点：覆盖面广，能发现更多来源页面。
- 风险：搜索结果不等于可用授权；缩略图、转载图、盗图和上下文错误较多。
- 使用：只作为发现入口。入选前必须回到原始页面核验来源、授权和上下文。

## AI 生成图片

- 适合：抽象概念、非事实示意图、情绪化封面、无法找到合适真实图片且不需要事实证据的场景。
- 优点：可按文章风格和版式定制，避免搬运版权不明的装饰图。
- 风险：不能证明事实，不能冒充真实现场、新闻图片、人物肖像、产品界面、报告图表或历史资料。
- 使用：只有搜索不足、版权风险过高或用户明确希望生图时才作为备选。输出中必须标记 `ai_generated: true`，并保留生成提示词或生成说明。

## 公众号场景建议

- 事实、产品、政策、报告文章：官方来源、原文图片、Wikimedia/Openverse 优先。
- 方法论、职场、商业评论：必要时用图库场景图，但图注不要伪装成事实证据。
- 热点、事故、争议、人物：谨慎使用全网图片，优先新闻机构授权图或官方图，不确定则不配。
- 教程、工具、软件：优先使用实际截图或官方文档截图，注意版本一致。
- 概念型封面或无法具象化的抽象主题：搜索不到合适可用图时，可考虑 AI 生图，但图注或内部记录应说明其为生成图。
